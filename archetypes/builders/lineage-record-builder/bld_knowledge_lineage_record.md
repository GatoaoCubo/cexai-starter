# Knowledge Card: lineage_record

## PROV-O Core Vocabulary
| Term | Definition | CEX Usage |
|------|-----------|-----------|
| prov:Entity | A thing with identity | knowledge artifact, document, dataset |
| prov:Activity | Something that occurs over a period | ingestion, synthesis, distillation |
| prov:Agent | A thing that bears responsibility | nucleus, tool, human curator |
| wasDerivedFrom(E2, E1) | E2 derived from E1 | knowledge_card derived from rag_source |
| wasGeneratedBy(E, A) | E was generated by activity A | artifact generated by F6 PRODUCE |
| wasAttributedTo(E, Ag) | E attributed to agent Ag | artifact attributed to N04 |
| used(A, E) | Activity A used entity E | ingestion activity used raw_document |

## Derivation Types
| Type | When to Use |
|------|------------|
| wasDerivedFrom | Target artifact built directly from source |
| wasGeneratedBy | Target artifact is output of a specific activity |
| wasQuotedFrom | Target contains verbatim content from source |
| wasRevisionOf | Target is an updated version of prior artifact |

## Curation Activity Labels
- `ingestion`: raw source -> chunked/indexed form
- `synthesis`: multiple sources -> unified knowledge card
- `distillation`: verbose artifact -> dense summary
- `validation`: artifact reviewed against quality gates
- `annotation`: artifact enriched with metadata

## When to Create a lineage_record
- After synthesizing a knowledge_card from multiple rag_sources
- After a FRACTAL_FILL or knowledge import wave
- When an artifact is derived from prior artifacts in a chain
- For compliance or reproducibility requirements
