## Phase 1: RESEARCH  
1. Define self-evolution objectives (e.g., adaptability, efficiency).  
2. Collect system performance metrics and failure modes.  
3. Analyze feedback mechanisms from prior iterations.  
4. Identify gaps in current governance policies.  
5. Design evolutionary triggers (e.g., error thresholds, performance drops).  
6. Simulate loop behavior using historical data.  

## Phase 2: COMPOSE  
1. Reference bld_schema_self_improvement_loop.md for structural requirements.  
2. Define feedback loop components (sensor, evaluator, modifier).  
3. Write sensor module to capture real-time system data.  
4. Implement evaluator using metrics from Phase 1.  
5. Code modifier to apply governance rules (see bld_output_template_self_improvement_loop.md).  
6. Integrate loop with existing system architecture.  
7. Embed versioning to track iterative changes.  
8. Add safety guards to prevent destabilizing updates.  
9. Finalize documentation per OUTPUT_TEMPLATE.md.  

## Phase 3: VALIDATE  
- [ ] Verify schema compliance (bld_schema_self_improvement_loop.md).  
- [ ] Test loop with synthetic failure scenarios.  
- [ ] Confirm feedback accuracy via metric cross-checks.  
- [ ] Ensure governance rules align with P11 Pillar.
