# GLM Cascade Routing -- Mode B Decompose on the Self-Hosted Box

## Executive Summary
Strong model thinks, cheap model types, deterministic tools govern. Industry:
cascade routing / model cascading with verifier fallback (FrugalGPT
planner-executor); CEX: 8F Mode B decompose. Stage 1 (F1-F4, claude-sonnet-4-6)
emits a self-contained prompt_package; Stage 2 (F6 ONLY) renders it on the box
(7 models); Stage 3 (F7-F8, zero-token Python) runs H01-H06 +
compile/commit/signal. Quality born in Stage 1, guaranteed in Stage 3: a weak
draft fails LOUD (live 2026-07-13: 3 glm-cpw calls, FAIL 5/6 H01, HEAD
untouched). Cost ~25-30% of premium solo; local glm-cpw F6 = electricity.

  frontier (quality-per-cost). Source: p07_bm_mission_bench.md "Per-Task
  Breakdown" row T1 + "Round-to-Round Variance" consistent cells.
  DGUARD/FACTUAL_SYNTHESIS_KINDS (cex_decompose.py) was OFF; -> warn (R-343).

## Anti-Patterns
- max_tokens<300 on reasoning models: budget burns in reasoning, content EMPTY.
- Trusting structural 6/6 as factual quality -- H01-H06 do not measure truth.
- Unknown model gaining reasoning duty: router fail-open resolves unregistered
  ids as full_8f (R-337); token-prefix collision on bare glm-5.2 (R-344). OPEN.
- signal-without-deliverable: decompose emitted complete 9.0 on a gate FAIL
  (hardcoded; R-343, fix in HARDEN_0713).
- Silent GLM->Claude fallback, undisclosed; declared fallback chain
  (gemini-flash-lite/qwen3) = legacy-passthrough fiction (R-343).
