Is CEXAI an agent framework?O CEXAI é um framework de agentes?
No. CEXAI is a typed knowledge system. Agent frameworks give you tools to build one agent. Think of it as the operating system, not the app.Não. O CEXAI é um sistema de conhecimento tipado. Frameworks de agentes dão a você ferramentas para construir um agente. Pense nele como o sistema operacional, não o aplicativo.
Do I need to know Python?Preciso saber Python?
No. CEX is designed for Claude Code -- you describe what you want in plain language, and the 8F pipeline handles the rest. The Python tools run automatically in the background. If you want to customize or extend, Python knowledge helps, but it is not required to get value on day one.Não. O CEX foi projetado para o Claude Code -- você descreve o que quer em linguagem natural, e o pipeline 8F cuida do resto. As ferramentas Python rodam automaticamente em segundo plano. Can I use this with GPT, Gemini, or local models?Posso usar isso com GPT, Gemini, ou modelos locais?
Yes. CEXAI is provider-agnostic. The default is Claude (highest quality), but every nucleus has a fallback chain configured in one YAML file. Swap to GPT-5 via Codex, Gemini via Google CLI, or run fully offline with Ollama. Your artifacts are the same regardless of which model produces them.Sim. O CEXAI é agnóstico de provedor. Troque para GPT-5 via Codex, Gemini via Google CLI, ou rode totalmente offline com Ollama. Seus artefatos são os mesmos independentemente de qual modelo os produz.
What is the difference between a "kind" and a "builder"?Qual a diferença entre um "kind" e um "builder"?
A kind is a typed artifact template (e.g., knowledge_card, agent, workflow). There are 318 kinds. A builder is the 12-file specification that teaches an LLM how to produce a specific kind. Think of it as: the kind is the blueprint, the builder is the factory that follows the blueprint.Um kind é um template de artefato tipado (ex.: knowledge_card, agent, workflow). Existem 318 kinds. Um builder é a especificação de 12 arquivos que ensina uma LLM a produzir um kind específico. Pense assim: o kind é a planta, o builder é a fábrica que segue a planta.

How You Drive ItComo Você Dirige Isso

One operator. No grid, no dispatch.Um operador. Sem grid, sem dispatch.

The factory that fabricated this repo runs multi-nucleus dispatch across parallel processes. This starter is leaner by design: the same 8F pipeline, driven by one operator -- you, or your agent -- in a single session.A fábrica que fabricou este repositório roda dispatch multi-nucleus em processos paralelos. Este starter é mais enxuto por design: o mesmo pipeline 8F, conduzido por um único operador -- você, ou seu agente -- numa única sessão.

$ python _tools/cex_bootstrap.py --check            # confirm your tenant checkout# confirma o checkout do seu tenant
$ python _tools/cex_doctor.py                       # repo-wide health check# verificação de saúde do repositório inteiro
$ python _tools/cex_8f_runner.py "<intent>" --dry-run --verbose  # preview a build# preview de um build
$ python _tools/cex_8f_runner.py "<intent>" --execute --verbose  # real build# build real
$ python _tools/cex_compile.py <path>               # compile after building# compilar após construir
$ python _tools/cex_score.py <path> --dry-run       # peer-review score (never self-assigned)# pontuação de peer-review (nunca autoatribuída)
