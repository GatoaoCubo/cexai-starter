# Orchestrator Type Definitions

## Purpose

Orchestration re-describes the same fields in free prose on every handoff
unless typed once. These types define the vocabulary; artifacts reference it.

## Enumerations

| Enum | Values | Meaning |
|---|---|---|
| NucleusId | n01..n07 | intelligence, marketing, engineering, knowledge, operations, commercial, orchestrator |
| WaveStatus | pending, running, complete, failed, skipped | Lifecycle of one dispatch wave |
| MissionPhase | planning, dispatching, monitoring, consolidating, done | Lifecycle of a whole mission |
| SignalType | complete, failed, partial, heartbeat | What a nucleus reports back |

## WavePlan

One execution wave: which nuclei run, dependencies, timeout.

| Field | Type | Required | Description |
|---|---|---|---|
| wave_id | string | yes | Unique id, e.g. `wave_1` |
| nuclei | list[NucleusId] | yes | Nuclei dispatched this wave |
| dependencies | list[wave_id] | no | Wave IDs that must be `complete` first |
| timeout_sec | int | yes | Max seconds before the wave is declared failed |
| status | WaveStatus | yes | Current execution state |

## HandoffSpec

Contract written before dispatch (renders as a `handoff` artifact, e.g. [[p12_ho_n07]]).

| Field | Type | Required | Description |
|---|---|---|---|
| mission | string | yes | Mission name, matches MissionState.name |
| nucleus | NucleusId | yes | Target nucleus |
| deliverables | list[string] | yes | Expected output paths |
| context_refs | list[string] | no | Artifact paths the nucleus must read first |
| decisions_ref | string or null | no | Path to a locked decision manifest |

## SignalPayload

Reported back after dispatch (renders as a `signal`, see [[p12_sig_admin_orchestration]]).

| Field | Type | Required | Description |
|---|---|---|---|
| nucleus | NucleusId | yes | Emitting nucleus |
| event | SignalType | yes | Completion category |
| score | float or null | no | Quality score if available |
| path | string or null | no | Primary artifact path |
| timestamp | datetime | yes | ISO 8601 emit time |

## ConsolidationReport

Produced after the final wave.

| Field | Type | Required | Description |
|---|---|---|---|
| mission | string | yes | Mission name |
| artifacts_verified / expected | int | yes | Confirmed on disk / declared in HandoffSpec |
| quality_avg | float or null | no | Mean quality; null if not yet scored |
| gaps | list[string] | no | Missing or below-floor artifact paths |

## Flow

`WavePlan` dispatches a `HandoffSpec` per nucleus -> nucleus returns a
`SignalPayload` -> final wave rolls signals into a `ConsolidationReport` ->
gaps feed back into the next `WavePlan`.
