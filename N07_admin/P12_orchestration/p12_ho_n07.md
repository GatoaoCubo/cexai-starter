# Handoff Example: N07 -> {{nucleus}} Dispatch

This is a worked example of what `p03_pt_n07_dispatch_handoff.md` renders. N07
writes one of these to `.cex/runtime/handoffs/{{mission}}_{{nucleus}}.md` (and
copies it to `{{nucleus}}_task.md` for boot-script consumption) before every
solo, grid, or decompose dispatch. Fill every `{{slot}}`; nothing ships unresolved.

```markdown
---
task: dispatch
from: n07
to: {{nucleus}}
mission: {{mission_codename}}
created: "{{iso_date}}"
status: active
related: [{{plan_id}}, {{decision_manifest_ref}}]
---

# Task for {{NUCLEUS_UPPER}} -- {{one_line_task_title}}

## Context (pre-loaded for you)
{{prior_state_summary -- what is already true/done; do not redo it}}

## DECISIONS (from user -- DO NOT re-ask)
{{decisions_block -- read from decision_manifest.yaml, or a defer-to-GDP note}}

## Relevant artifacts (READ before producing)
1. {{builder_iso_path}}
2. {{knowledge_card_path}}
3. {{prior_artifact_or_rule_path}}

## Expected output
| # | Path | Kind | Notes |
|---|---|---|---|
| 1 | {{output_path}} | {{output_kind}} | {{what_and_why}} |

## SCOPE FENCE
- ONLY: {{allowed_paths}}
- NEVER: {{forbidden_paths}}

## Gates
compile . doctor 0 FAIL . commit scoped to fence . signal on complete

## COMMIT
git add {{paths}}
git commit -m "[{{NUCLEUS_UPPER}}] {{mission_codename}}: {{summary}}"

## SIGNAL
python -c "from _tools.signal_writer import write_signal; write_signal('{{nucleus}}', 'complete', {{score}}, '{{mission_codename}}')"
```

## Handoff Rules

1. One handoff per nucleus per mission -- never bundle tasks for multiple nuclei.
2. Scope fence is mandatory -- the builder must not touch files outside it.
3. Commit and signal before any pause; a handoff without a signal is unfinished.
4. Quality floor: 8.0 for operational files, 9.0 for artifacts meant to ship.
